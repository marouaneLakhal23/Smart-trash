Absolument ! Pour votre rapport de stage, il est très pertinent d'inclure la structure de la base de données. Voici une description des tables et de leurs champs, basée sur les modèles définis dans votre fichier app.py :
Structure de la Base de Données (database.db - SQLite)
L'application utilise une base de données SQLite gérée via SQLAlchemy. Elle comprend les trois tables suivantes :
  1. Table user
    • Objectif : Stocker les informations des utilisateurs pouvant se connecter à l'application.
    • Champs :
      • id (Integer, Clé Primaire) : Identifiant unique de l'utilisateur.
      • username (String(80), Unique, Non Null) : Nom d'utilisateur pour la connexion.
      • password_hash (String(128), Non Null) : Hash sécurisé du mot de passe de l'utilisateur.
  2. Table bin
    • Objectif : Stocker les informations sur chaque poubelle intelligente.
    • Champs :
      • id (Integer, Clé Primaire) : Identifiant unique de la poubelle.
      • bin_number (String(50), Unique, Non Null) : Numéro ou identifiant unique de la poubelle (ex: "P-001").
      • location (String(200), Non Null) : Adresse ou description de l'emplacement de la poubelle.
      • current_level (Integer, Non Null, Défaut: 0) : Dernier niveau de remplissage connu (en pourcentage).
      • last_updated (DateTime, Défaut: Heure actuelle UTC) : Date et heure de la dernière mise à jour reçue pour cette poubelle.
      • last_emptied_timestamp (DateTime, Nullable) : Date et heure de la dernière fois où une collecte a été détectée (passage de >=80% à <=20%). Peut être NULL si aucune collecte n'a encore été enregistrée.
  3. Table history
    • Objectif : Enregistrer l'historique des événements où le niveau de remplissage a atteint un seuil critique (>= 80%).
    • Champs :
      • id (Integer, Clé Primaire) : Identifiant unique de l'entrée d'historique.
      • bin_id (Integer, Clé Étrangère vers bin.id, Non Null) : Identifiant de la poubelle concernée par l'alerte.
      • timestamp (DateTime, Non Null, Défaut: Heure actuelle UTC) : Date et heure exactes de l'enregistrement de l'alerte.
      • level (Integer, Non Null) : Niveau de remplissage (>= 80%) enregistré lors de l'alerte.
Relations :
  • Une relation "un-à-plusieurs" existe entre la table bin et la table history via le champ bin_id. Chaque poubelle (bin) peut avoir plusieurs entrées d'historique (history), mais chaque entrée d'historique ne concerne qu'une seule poubelle.
